<objective>
Build the `src/lib/broker/` module: an SSRF-hardened, PII-excluding, best-effort broker-page enrichment path that recovers the two fields Apollo genuinely lacks — renovationStatus and description — and a gap-fill-only merge that tags provenance. This is the security core of the phase: `agencyListingUrl` is Booli-sourced data pointing at an arbitrary third-party domain (a genuine new SSRF surface Phase 5's fixed-hostname allowlist cannot cover), and broker JSON-LD commonly embeds mäklare PII that must never enter storage.

Purpose: RESEARCH's primary recommendation — floor/brfName/balcony come from Apollo (Plan 01, no network); renovationStatus + description are the ONLY genuine broker-page dependency. Every hand-rolled alternative (regex HTML scraping, blocklist SSRF, raw JSON-LD passthrough) is a documented anti-pattern; this plan uses cheerio + `new URL()` + `dns.lookup` + explicit allow-list extraction. The module is pure/testable in isolation: fetch, guard, parse, and merge are separate functions with synthetic HTML fixtures. Plan 03 wires it into `analyzeUrl`.

Output: `url-guard.ts`, `fetch-broker-page.ts`, `parse-broker-page.ts`, `merge-listing-fields.ts` + tests + synthetic fixtures; `cheerio` installed.
</objective>
