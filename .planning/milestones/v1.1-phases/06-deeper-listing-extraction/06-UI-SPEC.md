**Field behavior notes (binding, derived from LSTG-03/04 + codebase precedent):**
- BRF name: extends the existing `brfName` slot in `ListingSummary` (`data.brfName ? <p>{data.brfName}</p> : "BRF-namn ej tillg√§ngligt"` ‚Äî already present, line 52-58 of `listing-summary.tsx`). Phase 6 only changes what CAN populate it (broker page as a second source after Booli/Apollo-state) ‚Äî the display contract is unchanged. Add the provenance caption underneath when the value is non-null.
- Floor, balcony, renovation status: NEW `MetricCard`-style entries appended to the existing metric grid (`grid grid-cols-2 gap-4 md:grid-cols-3`), styled identically to existing cards (Pris, Storlek, Rum, etc.) ‚Äî same `rounded-lg bg-warm-gray-50 p-4` container, same label/value typography.
- Description: NEW full-width prose block below the metric grid (not inside a `MetricCard` ‚Äî it's long-form text, not a short metric). Render inside a `rounded-lg bg-warm-gray-50 p-4` panel (consistent container language) with `text-sm leading-relaxed text-warm-gray-700` body, or "Ej tillg√§nglig" italic when absent.
- Every one of the 5 fields carries an inline provenance caption (`K√§lla: Booli` / `K√§lla: M√§klarens annons`) directly beneath its value ‚Äî EXCEPT when the value is "Ej tillg√§nglig" (no source to cite for a null field ‚Äî omit the caption in that case, matching `report-flags.tsx`'s "only cite when there's something to cite" discipline).
- LSTG-04's "never overwrite Booli data" rule is a data-layer contract, not a UI concern ‚Äî but the UI must make the provenance visibly checkable, which is why every populated field shows its source caption (this is the UI-observable proof that the fill-gaps-only rule held).
