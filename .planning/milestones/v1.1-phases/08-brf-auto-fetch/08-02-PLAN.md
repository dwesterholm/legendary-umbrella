<objective>
Build the source-acquisition layer that turns a Phase-6 `brfName` + listing geography into a fetched årsredovisning, without ever guessing a wrong BRF. Three modules:

1. `org-nr-resolver.ts` — resolves `brfName` (+ kommun/address from the listing breadcrumbs) to an organisationsnummer via a registry lookup, returning a discriminated `OrgNrResolution` where `high` confidence requires exactly-one-candidate AND geographic corroboration (per Pitfall 4 / Open Question 3). Includes strict Swedish org.nr format + Luhn validation before any string touches an outbound URL (ASVS V5).
2. `allabrf.ts` — the SSRF-guarded Allabrf fetch (name→org.nr candidates and org.nr→document), reusing the shared `resolveSafeExternalUrl` guard with a host allowlist, bounding document size, and stripping the iXBRL/HTML to text via Plan 01's `ixbrlToPlainText`.
3. `fetch-document.ts` — a phase-8-LOCAL fallback-tree walker (Allabrf as the single v1 rung; Bolagsverket bulk-feed DEFERRED, see Deferred Ideas) that logs every failed rung and throws-never-silently when exhausted, returning a `FetchedDocument` carrying text + org.nr + fiscal year + a not-most-recent staleness flag.

**Fallback-tree decision (resolved):** Build a phase-8-LOCAL walker in `fetch-document.ts` — option (b) from 08-RESEARCH.md Pattern 3 — NOT a generalization of `walkFallbackTree`. Rationale: `fallback-tree.test.ts` asserts on both the fixed literal `source` union AND the hardcoded `[booli-client]` log prefix and the "Alla Booli-källor" error message (verified by reading the test). Phase 8 needs a `[brf-source]` prefix and an "Alla årsredovisningskällor misslyckades" message; generalizing would break Phase 5's tested assertions. The local walker copies the exact discipline (per-rung logging, `rungs.length > 3` loud-fail, throw-on-exhaustion) without touching Booli's tested code.

Purpose: Provide confidence-gated org.nr resolution + document fetch as pure, unit-tested modules the Plan 03 action layer orchestrates (ENRICH-01 auto-fetch, ENRICH-02 fiscal-year surfacing + fall-through).
Output: three `src/lib/brf-source/*` modules + their tests.
</objective>
