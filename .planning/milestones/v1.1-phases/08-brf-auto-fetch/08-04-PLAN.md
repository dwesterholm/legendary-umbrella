<objective>
Wire the auto-fetch flow into the UI, per 08-UI-SPEC.md, keeping manual upload a first-class equally-prominent path. Four surfaces (2 new components, 2 modified) plus the page wiring:

1. New `BrfMatchConfirmation` (`brf-confirm.tsx`) — the "Stämmer detta med din bostad?" confirmation step showing org.nr + räkenskapsår (+ BRF name), a filled sage confirm button ("Ja, stämmer — analysera") and an equally-sized outline reject button ("Nej, ladda upp manuellt") that routes straight to manual upload.
2. New `BrfAutoFetchProgress` (`brf-auto-fetch-progress.tsx`) — the transient `auto_fetching` poller with its OWN short pre-step sequence ("Söker organisationsnummer…" → "Hämtar dokument…" → "Förbereder analys…"), copying `BrfProgress`'s visual pattern (not the component). Also add the `auto_fetching` step label handling to `brf-progress.tsx` per UI-SPEC (the two sequences stay distinct, never merged into one 6-step list).
3. Modified `BrfSection` — a `View` union extended with `"confirm"` / `"auto-fetching"`. On mount for an authenticated owner with a listing brfName, it calls `resolveOrgNrAction`: `high` → `confirm` view; `low`/`none`/fallThrough → the existing `upload` view with NO error banner (silent honest degrade). Confirm → `confirmAndAnalyze` → `auto-fetching` → the existing progress/result flow. Reject → `upload` with zero friction. The guest teaser stays the unconditional first branch (D-05).
4. Modified `BrfScoreCard` — a fiscal-year + provenance header row ("Räkenskapsår {year}", "Källa: Allabrf"/"Källa: Manuellt uppladdad") + an inline terracotta staleness caption when the year is not the most recent available.
5. `page.tsx` wiring — pass `listingData` (brfName/breadcrumbs/address) to `BrfSection` (it currently does not receive them) so resolution has its inputs; thread the fiscal year + source + staleness onto the score card.

Purpose: ENRICH-02 — confirmation before analysis, equally-prominent manual fallback, fiscal year shown + staleness-flagged.
Output: `brf-confirm.tsx`, `brf-auto-fetch-progress.tsx`, modified `brf-progress.tsx`/`brf-section.tsx`/`brf-score-card.tsx`, wired `page.tsx`, + component tests.
</objective>
