## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Raw Booli Apollo record → persisted DiscoveryCandidate | The scraped record may carry seller/occupant PII (names, phone numbers in description free text); the allowlist mapper is the boundary that must exclude it |
| discovery_jobs.results (JSONB) → results page read | A persisted candidate row is read back via discoveryCandidateSchema.safeParse (owner-only RLS from Phase 9) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-06 | Information Disclosure | PII leaking via raw description text persisted on the candidate | mitigate | Raw `description` is read ONLY as a local argument to `extractOrientationFromDescription` at toCandidate time and NEVER added to the returned object literal (no-spread construction + explicit no-description-key); only the derived `{facades, confidence}` (a bounded enum + a float, cannot carry PII) is persisted; the exact-key allowlist test is extended to assert `description` is not a candidate key |
| T-12-07 | Tampering | Weakening the allowlist test to accommodate new fields | mitigate | The exact-key `Object.keys(result).sort()` assertion is EXTENDED to the new key set (latitude/longitude/floor/orientation), never removed or loosened — the tripwire that guards DISC-07 stays intact and would fail on any accidental extra key |
| T-12-08 | Denial of Service | Non-finite latitude/longitude reaching downstream sun-path math | mitigate | `latitude`/`longitude`/`floor` pass through the existing `num()` coercion (rejects non-finite → null) at toCandidate time, so only finite numbers or null are ever persisted and later handed to computeSunExposure (which returns the unavailable sentinel on null) |
</threat_model>
