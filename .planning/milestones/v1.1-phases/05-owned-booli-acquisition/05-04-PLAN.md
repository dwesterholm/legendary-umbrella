## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → fetchListing | User-supplied URL crosses into the owned client → Apify browser |
| caller → fetchAreaListings | areaId/objectType crosses into a constructed URL |
| Apify render → Apollo extraction | Untrusted rendered payload parsed into a raw shape |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-10 | Tampering / Elevation of Privilege (SSRF) | `fetchListing(url)` | mitigate | `fetchListing` enforces `url.includes("booli.se/")` BEFORE constructing `startUrls`, preserving analyze.ts's allowlist — an arbitrary attacker URL never reaches the Apify headless browser. Asserted in tests (actorCall not called on bad URL). |
| T-05-11 | Tampering (parameter injection) | `buildTillSaluUrl` | mitigate | URL built with `URLSearchParams` (auto-encodes `areaId`/`objectType`), never manual string concatenation — mirrors `buildSlutpriserUrl`. |
| T-05-12 | Denial of Service | Apollo entity extraction | mitigate | Extraction uses null-tolerant coercion helpers + type guards (`typeof value === "object"` checks), never bare property access or `JSON.parse` on untrusted fields — a malformed entity yields nulls, never a throw that takes down the flow. |
| T-05-13 | Information Disclosure | client.ts logging | mitigate | `[booli-client]` logs carry only errors + source/rung/health; never the token or client instance. |
| T-05-14 | Spoofing (silent paid-actor default) | rung wiring | mitigate | The paid actor is wired ONLY as rung 3 inside `walkFallbackTree`; ACQ-03 surfaces `source`/`rung`/`health` so a paid-actor hit is observable, not the silent default it is today. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages (`apify-client` already a dependency). No slopcheck checkpoint required. |
</threat_model>
