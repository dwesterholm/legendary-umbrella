<objective>
Build the two-pass vision pipeline (Haiku pre-filter → Sonnet deep pass) that turns a candidate's capped image set into hedged, image-cited condition claims — with the incremental vision cost cap, confidence-threshold suppression, PII-ignore prompt, and GDPR-safe error logging — plus the RUN_LLM_EVALS-gated eval harness and rubric. Wire the vision pass into the discovery job lifecycle without ever feeding its output into the deterministic scorer.

Purpose: This is the DISC-04 core. The two-pass gate (conditional second AI call) is the one genuinely-new orchestration in this codebase; everything else is a mechanical extension of extract.ts's `messages.parse` + `zodOutputFormat` + coded-error discipline with `image` content blocks. Cost discipline (Haiku-triage-then-Sonnet, incremental cap-before-spend) is what keeps vision under budget.

Output: `vision.ts` (orchestration), `vision-prompt.ts` (PII-ignore prompts), job wiring, `evals/vision.eval.ts` + label example. The live 20–30-listing validation RUN and the one live API smoke are operator-deferred (see verification).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → Anthropic (Haiku/Sonnet) | Server-only vision calls; API key must never reach the browser |
| image/description content → model | Untrusted scraped image/description content is model input (prompt-injection surface) |
| model output → persisted results | Hedged claims persist into `discovery_jobs.results`; PII or fabrication would violate the locked constraint |

| T-11-06 | Information Disclosure | PII in photos | mitigate | Explicit people/personal-document ignore instruction in BOTH prompts + a schema (Plan 01) with no field capable of referencing a person; eval's zero-hallucination hard gate (100%) verifies |
