## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| enrich action тЖТ Riksbank/SCB | Server-side calls cross into two external, untrusted, keyless HTTP APIs (fixed hosts) |
| server-derived region тЖТ SCB query body | A l├дn code derived from resolveGeo() is interpolated into the PxWebApi request |
| Supabase client тЖТ macro_snapshots | First non-owner-scoped shared table; RLS default-deny risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure / Denial | macro_snapshots RLS | mitigate | Explicit SELECT + INSERT + UPDATE policies gated on `auth.uid() is not null` in 006 migration тАФ absent policy = total lockout (RLS default-deny), not a leak, but must be deliberate (RESEARCH Pitfall 2). Data is non-PII publicly-sourced statistics; no confidentiality reason to restrict further (A4). |
| T-07-02 | Tampering (SSRF) | region code тЖТ SCB query body | mitigate | `isValidLanCode` allowlist (l├дn `01`-`25` + aggregates) before any region value reaches the PxWebApi body; region is always server-derived from `kommunCode.slice(0,2)`, never client free-text (T-03-06 precedent, ASVS V5). Riksbank/SCB are fixed hosts тАФ no dynamic host construction. |
| T-07-03 | Denial of Service | Riksbank/SCB fetchers | mitigate | Every fetch wrapped in try/catch + Zod `.safeParse`; null-on-any-failure (mirrors `fetchScbTable`) so a malformed/adversarial payload can never throw uncaught into the enrich pipeline. Read-through TTL cache (24h) bounds live calls to at most once per window system-wide, honoring the SCB 30-calls/10s budget (Pitfall 4). |
| T-07-04 | Tampering (cache poisoning) | macro_snapshots payload | accept | Payload is written only by the server action after Zod-normalization of a fixed-host response; the read guard `safeParseMacroData` re-validates on read (Plan 03). Any authenticated user can upsert (A4), but the value is deterministic public data тАФ a poisoned row self-heals on the next stale-refresh and carries no PII. Low-value target for a solo-dev product. |
| T-07-SC | Tampering | npm/pip/cargo installs | n/a | No packages installed this phase (RESEARCH Package Legitimacy Audit тАФ native fetch + existing zod only). Gate does not apply. |
</threat_model>
