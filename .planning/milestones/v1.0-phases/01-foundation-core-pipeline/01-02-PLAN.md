    1. Create Zod schemas (`src/lib/schemas/listing.ts`):
       - `scraperOutputSchema`: Flexible schema for Apify output. All fields optional except `url`. Use `.passthrough()` to allow unknown fields. Fields: url, address, price, livingArea, rooms, monthlyFee, buildYear, floor, brfName, propertyType, neighborhood.
       - `listingDataSchema`: Our internal model with required core fields (url, address, price, livingArea, rooms) and nullable optional fields (monthlyFee, buildYear, brfName). Includes calculated `prisPerKvm` (number).
       - Export types: `ScraperOutput`, `ListingData`
       - NOTE: Exact Apify field names are unconfirmed (RESEARCH open question #1). Use the names from RESEARCH.md as best guess. The Zod schema's `.passthrough()` and `.optional()` pattern ensures graceful handling even if field names differ. The first real scrape will reveal the actual field mapping.

    2. Create Apify scraper utility (`src/lib/apify/booli-scraper.ts`):
       - Import `ApifyClient` from `apify-client`
       - Initialize client with `process.env.APIFY_API_TOKEN`
       - Export `scrapeBooli(url: string)` function:
         - Calls `client.actor("lexis-solutions/booli-se-scraper").call({ startUrls: [{ url }] }, { waitSecs: 30 })`
         - Retrieves dataset items via `client.dataset(run.defaultDatasetId).listItems()`
         - Returns `items[0]` or throws descriptive error if no results
         - Wrap in try/catch: on timeout or actor error, throw with Swedish-friendly error message

    3. Create utility helpers (`src/lib/utils.ts`):
       - `formatSEK(amount: number): string` -- formats as "1 234 567 kr" with Swedish thousand separators
       - `calculatePrisPerKvm(price: number, area: number): number` -- returns price / area rounded to nearest integer
       - `cn(...classes)` -- Tailwind class merger (if not already from shadcn init)

    4. Create server action (`src/actions/analyze.ts`):
       - `"use server"` directive
       - Export `analyzeUrl(formData: FormData)`:
         a. Extract `url` from formData
         b. Validate URL contains "booli.se/" -- if not, return `{ error: "Ange en giltig Booli-lank" }`
         c. Call `scrapeBooli(url)`
         d. Parse with `scraperOutputSchema.safeParse(rawData)`
         e. Transform to internal listing data: map fields, calculate prisPerKvm
         f. If authenticated user: insert into `analyses` table via Supabase server client, redirect to `/analysis/{id}`
         g. If guest (no user):
            - Check `guest_analysis_done` cookie
            - If cookie exists: redirect to `/login?reason=guest-limit`
            - If no cookie: set cookie (httpOnly, 30 day maxAge), return data directly
         h. On scraper error: return `{ error: "Kunde inte hamta data fran Booli. Forsok igen." }`
         i. On partial parse: return `{ data: partialData, partial: true, missingFields: [...] }` with whatever fields were extracted
  </action>
