<objective>
Implement the deterministic, provider-agnostic core that turns an extraction result into an auditable A–F grade. Four pure modules: the Zod extraction schema + null-tolerant normalizer (mirrors `listing.ts`), the rule-based scorer (D-08, no Claude), the sanity-range confidence guardrail (D-10), and the cost calculator. This plan makes the RED tests from Plan 01 GREEN.

Purpose: This is the trust core. The grade MUST be reproducible (D-08) and publicly explainable (D-09 methodology page in Plan 05 reads the SAME threshold constants defined here). Claude only supplies numbers; this code decides the grade. Sanity bands force confident-but-wrong values to the surface (D-10).
Output: `src/lib/schemas/brf.ts`, `src/lib/brf/score.ts`, `src/lib/brf/sanity.ts`, `src/lib/brf/cost.ts` — all consumed by the Plan 04 server action and Plan 05 UI/methodology page.

</objective>
