<objective>
Build the extraction layer and the `analyzeBrf` server action that wires it all together: auth-gate (D-05) → store PDF privately → content-hash skip (D-06) → ONE Haiku call with citations + caching → normalize → sanity-check → deterministic grade → persist with status + cost. Also the `correctBrfField` action for D-12 inline edits that re-run normalize+sanity+score WITHOUT re-calling Claude.

Purpose: This is the spine — the project's first Claude integration. It must keep the API key server-only (D-05), never re-bill the PDF on retry (cache_control), enforce the <5 SEK budget, and surface low-confidence/refusal cases to the manual-entry path rather than returning silent wrongness.
Output: `prompt.ts`, `extract.ts`, `storage.ts`, `analyze-brf.ts`, plus next.config.ts/.env.example updates.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → server action | Untrusted File upload (type/size/encryption) crosses into the server |
| server → Anthropic API | API key must never cross back to the browser |
| user → analyses row / bucket | Cross-user access must be impossible |
| extraction → logs | Financials/PII must never enter logs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-09 | Information Disclosure | `ANTHROPIC_API_KEY` | mitigate | SDK instantiated only in server-side `extract.ts`; never `dangerouslyAllowBrowser`; env var server-only (no NEXT_PUBLIC_); verified in Task 1 grep + Task 3 smoke test |
| T-02-10 | Elevation of Privilege | `analyzeBrf` / analyses row | mitigate | Hard server-side auth gate (D-05) + ownership check on analysisId + RLS UPDATE policy (Plan 02) |
| T-02-11 | DoS / Tampering | File upload | mitigate | Server-side type=application/pdf + size ≤20 MB validation before storage; reject encrypted PDFs; Files API for large scans to avoid 32 MB cap failure |
| T-02-12 | Information Disclosure (PII) | extraction logs | mitigate | Log only content hash + token usage; NEVER raw bytes, financials, or source quotes (GDPR, AI-SPEC §7) |
| T-02-13 | DoS (budget) | Claude spend | mitigate | cache_control on document block + bounded retries + per-analysis 5 SEK cost cap + login gate; content-hash skip on identical re-upload |
| T-02-14 | Tampering | model-produced grade leak | mitigate | Extraction schema has no grade field; action computes grade via computeBrfGrade only (D-08) |
</threat_model>
