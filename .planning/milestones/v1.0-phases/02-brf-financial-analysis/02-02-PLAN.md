<objective>
Create migration `002_brf.sql` that adds the BRF persistence layer: a private `brf-pdfs` Storage bucket with per-user RLS, the `brf_*` columns on the existing `analyses` table, and the UPDATE RLS policy that 001 omitted (the server action and inline-edit re-score both write via `.update()`). Then push the schema to the live Supabase database.

Purpose: PDFs must be stored privately (GDPR — they can contain styrelse personal data; no public URLs). Extraction results, status (D-13), cost, content hash (D-06 replace-cache), and the scanned flag (D-14) all persist on the analyses row. The UPDATE policy is load-bearing: without it, status writes and D-12 corrections silently fail under RLS.
Output: `supabase/migrations/002_brf.sql` applied to the live database.
</objective>
