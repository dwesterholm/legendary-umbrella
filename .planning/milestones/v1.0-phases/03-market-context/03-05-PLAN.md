<objective>
Build the server-action spine that orchestrates Phase 3: `enrichMarketContext(analysisId)` — auth → ownership → resolve geo → fetch both external sources → run the deterministic compute → persist `price_data` and `area_data` INDEPENDENTLY with status/source/cost.

Purpose: This is the controller tying the Plan 03 (SCB/geo) and Plan 04 (sold/compare) cores to the Plan 02 columns. It mirrors the Phase 2 `analyze-brf.ts` spine exactly (auth gate → ownership → external → cost gate → persist → terminal status). The critical Phase 3 property (Success Criterion 3, D-08): each panel persists independently, so a failing/thin sold source still yields area_data, and a geocoding miss / SCB gap still yields price_data. Older analyses lacking coords degrade to kommun-baseline, no backfill (RESEARCH Open Q3).

Honest reasons (HIGH-1 / HIGH-3): this action OWNS the distinction between a dead source and a thin area. A sold-source fetch/parse FAILURE (the throw / sentinel from Plan 04's fetchSoldComps) → persist price_data with `reason: "source_unavailable"`. A REAL area query that returned few comps → compare.ts already tags `reason: "thin"`. A listing whose own prisPerKvm is 0/falsy → set/keep `reason: "listing_pris_okand"` (compare.ts guards this, and this action must NOT call compare with a 0 listing figure expecting a real headline). These three never collapse into one — that is the fix for the silent PRICE-01 / false −100% failures.
Output: enrich-market-context.ts.

Implements: D-08 (partial-data, independent persistence), D-09 (source+freshness labels persisted for the trust display), PRICE-01 + AREA-01 orchestration.
</objective>
