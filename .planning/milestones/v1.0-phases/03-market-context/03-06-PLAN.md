<objective>
Wire the two market-context panels into the analysis page, replacing the "Prisjämförelse" and "Områdesstatistik" *Kommer snart* placeholders with real data — reusing the Phase 2 BRF visual language (confidence badge + source reveal + warm palette). Each panel renders and degrades INDEPENDENTLY so the page always loads (Success Criterion 3, D-08).

Purpose: This is the user-facing payload of Phase 3. The price card surfaces the ±% headline (D-04), the comp-list "receipt" (D-05), sample size + tier (D-01/D-05), distribution (D-05), confidence (D-09), source+freshness label and the "ej värdering" disclaimer (D-09). The area card surfaces the four SCB metrics (D-07) with the geo-level and "Källa: SCB" label. CRITICALLY, the price card must present THREE+ distinct honest states, never collapsing them into one (the HIGH-1 dishonesty fix): (a) a genuine comparison; (b) thin data — a REAL area query returned few/zero comps ("för få försäljningar"); (c) source_unavailable — we could not reach/parse the sold-price source at all ("prisjämförelse ej tillgänglig"); (d) listing_pris_okand — the listing's own pris/kvm is missing so no ±% is computable (HIGH-3). A dead source presented as merely sparse is a lie that silently leaves PRICE-01 unmet. A human-verify checkpoint confirms the real rendered behavior including these partial-data states AND gates PRICE-01 completion on a REAL live comparison (not the mock fixture).
Output: price-comparison-card.tsx, area-stats-card.tsx, market-context-section.tsx, and the modified analysis page.

Auth posture (HIGH-2 resolution — owner-only): Phase 3 market context is an AUTHENTICATED-OWNER feature by inheritance. Per src/actions/analyze.ts:96-124, a guest's analysis is returned in-memory and NEVER persisted (no row, no id), and src/app/(app)/analysis/[id]/page.tsx only renders for a persisted row. Therefore the market panels — which read persisted price_data/area_data — are unreachable for guests by construction. This plan does NOT add a guest read-only state and does NOT add a separate login wall; it simply renders the owner's persisted data and the auth-gated enrich trigger. This is consistent with how guest analyses already behave (the whole detail page is owner-only), and with Plan 05's hard auth gate on enrichMarketContext.

Implements: D-01, D-04, D-05, D-06, D-07, D-08, D-09 (display side); PRICE-01 + AREA-01 user-visible criteria.
