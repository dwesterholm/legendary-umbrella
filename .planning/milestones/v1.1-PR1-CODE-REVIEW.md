## Blockers (fix before merge)

### BL-1 · SSRF: IPv6 classifier misses ULA / IPv4-mapped / unspecified
`src/lib/broker/url-guard.ts:66-69` — `isUnsafeAddress` only rejects `::1` and `fe80:`.
`fc00::/7`, `::`, and **IPv4-mapped IPv6** (`::ffff:169.254.169.254`, `::ffff:10.x`,
`::ffff:127.0.0.1`) pass as "safe public." `dns.lookup` can return family-6 (it returned
`::1` for `localhost` on the review box), and the socket is pinned to that address — a
**live SSRF to cloud metadata / internal hosts**, exactly what the guard's doc comment
claims to block. Verified by direct execution. Fix: parse IPv6 to bytes, unwrap mapped
`::ffff:a.b.c.d` back through `isPrivateIPv4`, block `::` + `fc00::/7`; add regression
tests. (This guard is shared by the BRF fetch path too, so the fix covers shard 4.)

### BL-2 · No-prediction: banned-phrase list never runs at runtime
`src/lib/report/banned-phrases.ts` — `BANNED_PREDICTIVE_PHRASES` is imported **only** by
its test, never by `synthesize.ts`/`generate-report.ts`. The "tertiary MACRO-02
enforcement layer" the file documents does not exist in the running system; a model
response like "priserna kommer stiga" satisfies the schema, is never scanned, and is
persisted + rendered. Fix: scan `leadSynthesis` + all `claims[].text` against the list
after `reportSchema.parse` and treat a hit like a refusal (don't persist).

### BL-3 · Freshness: SCB macro queries pin stale-by-design date literals
`src/lib/market/macro.ts:235` (`time: ["2026M05"]`) and `:254` (`["2024","2025"]`) are
fixed literals. The CPIF figure is already ~2 months behind and only rots further; once
SCB stops offering that exact period the headline "aktuella nyckeltal" silently blanks or
staled with no signal and no code path that advances it. Fix: PxWeb `filter: "top"` time
selection (top 1 for inflation, top 2 for regional so the `lastNonNull` fallback survives).

### BL-4 · BRF: cache-hit path leaves status wedged at `auto_fetching`
`src/lib/brf/run-extraction.ts:248-253` — on a D-06 content-hash cache hit,
`runBrfExtraction` returns `ok:true` **without** advancing `brf_status` to `done`. The
client's only success signal is the `brf_status` poller, so it spins to a false-failure
timeout, and the CAS acquire predicate (excludes `auto_fetching`/`done`) is then
**permanently wedged** at "hämtning pågår redan" — reachable via the resolve→confirm→
timeout→re-confirm retry the CAS exists to protect. Fix: write `brf_status="done"` before
the cache-hit return (covers both `confirmAndAnalyze` and manual `analyzeBrf`).

## Cross-cutting theme — read-path image/fetch guards
Three findings share one root cause: **write-time allowlists not re-applied on the
read/render path.** Worth fixing together.
- BL-1 (SSRF read path, broker/BRF fetch).
- Shard-1 WR-03: `claimVisionSlice` sends persisted image URLs to Anthropic without
  re-running `isAllowedImageHost` (`vision.ts`/`job.ts`).
- Shard-5 WR-03: the rendered thumbnail array `imageUrlsUsed` is the one URL field with
  no read-path allowlist `.transform`, unlike `imageUrls`.
A tampered/corrupted/migration-bugged row bypasses the codebase's own defense-in-depth in
each case. Add the `.filter(isAllowedImageHost)` re-check on every read path that fetches
or renders a persisted URL.
