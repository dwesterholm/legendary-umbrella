<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SCB PxWebApi → server | Untrusted external json-stat2 response crosses into normalize (Zod) |
| listing lat/lng → SCB query | Resolved region codes used to build the query URL/body server-side |
| persisted area_data → RSC | Stored jsonb re-validated on read before render |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering (SSRF) | scb.ts query builder | mitigate | Build the json-stat2 query body server-side from validated lat/lng-derived region codes + whitelisted table ids; never interpolate user free-text into the SCB URL (RESEARCH Security) |
| T-03-07 | Tampering | normalizeScbOutput | mitigate | Permissive `.passthrough()` + null-tolerant coercion: malformed/partial SCB JSON falls back to null, never throws (RESEARCH Pattern 3, Pitfall 3) |
| T-03-08 | Denial of service (rate/cost) | SCB fetch | mitigate | Cache by (table, region, year) — SCB updates yearly; warm cache → zero calls; respect 30 calls/10s to avoid HTTP 429 (Pitfall 4). SCB is free → no monetary cost |
| T-03-09 | Information disclosure | persisted area_data | mitigate | safeParseAreaData re-validates on read; area_data lives under the existing per-user RLS (Plan 02) |
| T-03-SC | Tampering | @turf install | mitigate | [ASSUMED] packages verified via `npm view` (Turfjs org) behind a non-auto-approvable human checkpoint before install (Task 1) |
</threat_model>
