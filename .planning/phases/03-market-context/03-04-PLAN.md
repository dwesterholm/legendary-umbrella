<objective>
Build the PRICE-01 deterministic core (GREEN for the compare/cost RED tests from Plan 01): fetch comparable sold prices from the source chosen in the Plan 01 spike, normalize them null-tolerantly, and compute the ±% comparison, tier, trend, distribution, sample size, and confidence — all as pure arithmetic (D-08/D-09 trust model — NEVER an LLM).

Purpose: PRICE-01 display is gated on the Plan 01 spike (RESEARCH Open Q1). With the source decided, this plan delivers the source-isolating fetch interface plus the deterministic comparison engine. The comparison is statistics, not a valuation verdict (D-04/D-09). pris/kvm is the baseline normalization (D-03); rich attrs (floor/balcony/avgift) layer on ONLY where the chosen source exposes them — the comparison never blocks on them. Plan 05 orchestrates and persists.

Honest-state discriminator (HIGH-1 / HIGH-3): PriceData carries an explicit `reason` field so the three failure modes are NEVER collapsed: `ok` (a real comparison), `thin` (a REAL area query returned ≤ threshold comps), `source_unavailable` (the source could not be reached/parsed — set by Plan 05's catch, not by compare.ts), and `listing_pris_okand` (the listing's own pris/kvm is 0/null so a ±% is not computable). This is the fix for the "dead source displayed as sparse" dishonesty and the false "−100 % under snitt" headline.
Output: sold-schema.ts (Task 1), compare.ts + cost.ts (Task 2), sold-source.ts (Task 3).

Implements: D-01 (tiered fallback building→neighborhood→wide with explicit tier label), D-02 (24-month window + trend), D-03 (pris/kvm-first, rich attrs optional), D-04 (±% headline), D-05 (comp list + sample size + tier + distribution), D-08 (thin data → honest marker, never crash), D-09 (confidence from sample size + tier).
