<objective>
Close the two gaps the 13-03 live smoke surfaced (13-SMOKE-FINDINGS.md) so a real multi-area discovery run VISIBLY progresses within the client poll window — the "finishes in-window WITH visible progress" half of DXUX-01 that only a live run could expose. 13-01 (backend area parallelism + area-page wait ceiling) and 13-02 (two-tier timeout + status labels) already shipped; this plan fixes the three remaining levers: (1) the client status read is blocked behind the long in-flight tick, (2) vision enrichment writes progress only once at the very end, and (3) vision detail-page fetches use the unbounded 240s/3-retry default.

Purpose: DXUX-01 is not met if the badge stays "I kö" for 8+ minutes while the server is genuinely working. The soft-notice (13-02) was the ONLY thing updating during the long vision tick — the badge and counter never moved.

Output: A decoupled status read, incremental processed_count writes during enrichment, and a bounded per-candidate detail render — with the /analyze path and the D-03 cost-cap invariant untouched.
</objective>
